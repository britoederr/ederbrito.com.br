## Overview

This project is the platform behind `ederbrito.com.br`: a fully self-operated Kubernetes environment running on OCI's Always Free tier, at zero infrastructure cost.

The goals were practical and intentional: deploy a real, production-grade platform using the same tools I work with daily at Avalara (Kubernetes, Cilium, Terraform, Prometheus/Grafana/Loki), while proving it can be done sustainably on free-tier compute.

---

## Architecture

The platform runs on **Oracle Kubernetes Engine (OKE)** using two `VM.Standard.A1.Flex` ARM instances (2 OCPU / 12 GB each), Oracle's always-free compute tier. Network architecture follows a hub-and-spoke VCN design with public subnets for the API and load balancer, and private subnets for worker nodes with NAT and service gateways.

Traffic flows:

1. DNS (`ederbrito.com.br` + subdomains) → OCI Load Balancer (Cilium Gateway)
2. Gateway API `Gateway` → `HTTPRoute` → application / observability Services
3. TLS terminated at the gateway via cert-manager + Let's Encrypt (HTTP-01)
4. Observability subdomains (`grafana.`, `prometheus.`, `jaeger.`, `loki.`, `hubble.`) served through the same gateway

---

## Infrastructure as Code

All infrastructure is provisioned with **Terraform**:

- **VCN and networking** — subnets, NAT gateway, service gateway, security lists
- **OKE cluster** — BASIC cluster with ARM node pool; Flannel at create, then exclusive Cilium via Helm
- **OCI DNS** — A records for apex and all observability subdomains, dynamically resolved from the Cilium Gateway load balancer IP using Terraform's Kubernetes provider

Remote state is stored in **OCI Object Storage**. The two Terraform stacks (cluster and DNS) are decoupled so DNS can be updated independently without touching the cluster.

---

## CI/CD

A **GitHub Actions** pipeline automates the full provisioning lifecycle:

1. `terraform plan` and `terraform apply` for the OKE cluster and networking (apply on push only)
2. OCI CLI authenticates, fetches kubeconfig, and bootstraps Helm releases (Cilium + Hubble, cert-manager)
3. `kubectl apply` deploys the Kubernetes manifests (Gateway, Certificate, issuer, observability stack)
4. A second Terraform run applies DNS changes, reading the live gateway LB IP from the cluster

The pipeline triggers on changes under `sre/common/terraform/` and `sre/common/kubernetes/` so infrastructure PRs can be reviewed and validated before apply.

## Observability Stack

The cluster runs a full observability suite, accessible via dedicated subdomains and protected by the same TLS setup:

- **Prometheus** — Metrics collection and alerting
- **Grafana** — Dashboards and visualization
- **Loki** — Log aggregation
- **Jaeger** — Distributed tracing
- **Hubble** — Cilium network flow visibility (UI + Relay)

This mirrors the production observability patterns I design and operate at Avalara, applied here at a personal scale.

## Key Technical Decisions

**Why OCI Always Free?** The `VM.Standard.A1.Flex` shape provides enough compute for a real multi-node cluster without cost. This removes any incentive to cut corners on the platform layer — the goal was a legitimate production setup, not a dev sandbox.

**Why Cilium instead of Istio?** For a single-service site, a full service mesh was mostly an expensive ingress layer. Cilium provides eBPF networking, NetworkPolicy, and Hubble for L3/L4/L7 flow visibility; Gateway API covers ingress and TLS without sidecar overhead on Always Free nodes.

**Why Terraform + GitHub Actions instead of a managed PaaS?** Full control over the infrastructure lifecycle, a real GitOps-style workflow, and a meaningful artifact to demonstrate end-to-end IaC ownership.

## Technologies Used

- **Cloud:** Oracle Cloud Infrastructure (OCI), Always Free tier
- **Kubernetes:** OKE (BASIC), ARM nodes (`VM.Standard.A1.Flex`), v1.36
- **CNI / networking:** Cilium + Hubble (Helm)
- **Ingress:** Cilium Gateway API
- **TLS:** cert-manager v1.19.2, Let's Encrypt
- **IaC:** Terraform (OCI + Kubernetes providers), OCI Object Storage backend
- **CI/CD:** GitHub Actions
- **Observability:** Prometheus, Grafana, Loki, Jaeger, Hubble
- **DNS:** OCI DNS (managed via Terraform)
